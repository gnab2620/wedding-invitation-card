# Hướng dẫn Thiết lập RSVP với Supabase

Để chức năng RSVP hoạt động trên GitHub Pages, bạn cần thực hiện các bước sau để thiết lập cơ sở dữ liệu trên Supabase.

## Bước 1: Tạo tài khoản và dự án
1. Truy cập [Supabase](https://supabase.com) và đăng ký tài khoản miễn phí.
2. Tạo một dự án mới (New Project). Đặt tên tùy ý (ví dụ: `wedding-invitation`).
3. Đợi vài phút để dự án được khởi tạo.

## Bước 2: Tạo bảng dữ liệu (RSVPs Table)
1. Trong menu bên trái, chọn **SQL Editor**.
2. Nhấn **New Query**.
3. Copy đoạn code bên dưới và nhấn **Run**:

```sql
create table rsvps (
  id bigint generated by default as identity primary key,
  name text not null,
  guests integer default 1,
  message text,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- Enable Row Level Security
alter table rsvps enable row level security;

-- Allow anyone to insert (public submissions)
create policy "Allow public insert" on rsvps for insert with check (true);

-- Allow anyone to read (for your own tracking)
create policy "Allow public read" on rsvps for select using (true);
```

## Bước 3: Lấy thông tin API
1. Vào **Project Settings** (biểu tượng bánh răng) > **API**.
2. Tìm **Project URL** (ví dụ: `https://xyz.supabase.co`).
3. Tìm **anon public** key (một chuỗi ký tự dài).

## Bước 4: Cập nhật mã nguồn
1. Bạn **không cần** chỉnh sửa file `script.js` trên máy tính bằng mã thật.
2. Mã bảo mật sẽ được tự động điền vào thông qua GitHub Actions (xem Bước 5).
3. Nếu bạn muốn chạy thử ở máy tính cá nhân, bạn có thể tạm thời điền mã vào `script.js` nhưng hãy nhớ **xóa đi** trước khi commit để tránh lộ mã.

## Bước 5: Cấu hình GitHub để Deploy an toàn
Để trang web chạy được trên GitHub Pages mà không cần đẩy mã bí mật lên Git, bạn cần làm như sau:

1. Vào Repository của bạn trên GitHub.
2. Chọn **Settings** (cài đặt) > **Secrets and variables** > **Actions**.
3. Nhấn **New repository secret** để thêm 2 mã sau:
   - Name: `SUPABASE_URL` | Value: (Dán URL của bạn vào đây)
   - Name: `SUPABASE_ANON_KEY` | Value: (Dán Anon Key của bạn vào đây)
4. Sau khi thêm xong, mỗi khi bạn `git push`, GitHub sẽ tự động điền các mã này vào trang web một cách an toàn.

## Bảo mật & Riêng tư
- **Tại sao mã không ẩn hoàn toàn?** Vì bạn dùng GitHub Pages (trang tĩnh), trình duyệt của khách cần mã API để gửi dữ liệu về Supabase. Đây là điều bắt buộc.
- **Làm sao để an toàn?** Chúng ta sử dụng **Row Level Security (RLS)** sẵn có của Supabase (đã cấu hình ở Bước 2). RLS đảm bảo rằng dù ai đó có lấy được mã API, họ cũng chỉ có thể "Gửi thông tin" chứ không thể xóa hoặc xem lén danh sách khách mời của bạn nếu bạn không cho phép.

## Cách xem và quản lý danh sách khách mời trên Supabase

Bạn có thể theo dõi danh sách người tham dự bất cứ lúc nào bằng cách:

1. **Đăng nhập** vào [Supabase Dashboard](https://supabase.com/dashboard).
2. Chọn dự án của bạn (ví dụ: `wedding-invitation`).
3. Ở menu cột bên trái, nhấn vào biểu tượng **Table Editor** (hình ô lưới).
4. Chọn bảng `rsvps`. Toàn bộ thông tin khách mời đã đăng ký sẽ hiện ra ở đây.

### Các tính năng hữu ích:
- **Sắp xếp/Lọc**: Bạn có thể nhấn vào tiêu đề cột (ví dụ: `created_at`) để sắp xếp theo thời gian mới nhất.
- **Xuất file Excel (CSV)**: 
  - Nhấn vào nút **Export** ở góc trên bên phải bảng.
  - Chọn **Export as CSV**. Bạn có thể mở file này bằng Excel để in ấn hoặc quản lý.
- **Xóa/Sửa dữ liệu**: Bạn có thể chuột phải vào từng dòng để xóa nếu có thông tin rác hoặc khách mời thay đổi ý định.
- **Thống kê nhanh**: Bạn có thể chạy câu lệnh sau trong **SQL Editor** để biết tổng số khách:
  ```sql
  select sum(guests) as tong_so_khach from rsvps;
  ```
